## Design decisions

- **Real folder mirroring**: `showDirectoryPicker()` gives a `FileSystemDirectoryHandle`. The sidebar is a recursive traversal of the real folder — no virtual tree, no JSON index. Folders and files on disk are exactly what the sidebar shows.
- **Markdown stored as `.md` files**: notes are plain `.md` files, editable by any editor (Obsidian, VS Code, etc.) outside the app. No proprietary format.
- **Config sidecar**: `_notes.config.json` at the vault root stores UI prefs (last-open path). Excluded from the sidebar tree. Small enough to be written fire-and-forget.
- **Editor mode**: textarea (left) + live markdown preview (right), always visible. 50/50 default split, draggable divider. Split ratio persisted in config.
- **Markdown renderer** (`md.js`): ~100-line zero-dependency renderer. Covers headings, bold/italic/strikethrough, inline code, fenced code blocks, blockquotes, ordered/unordered/task lists (`- [ ]` / `- [x]`), horizontal rules, links. Front matter stripped before render.
- **Auto-save**: 750 ms debounce after last keystroke, same cadence as kanban. "Saved" badge fades after 2 s. Preview updates on every keystroke (synchronous, fast enough for any note size).
- **Delete**: `✕` button in editor header (hidden until a note is open). Confirms, permanently deletes via `removeEntry()`, clears editor, removes from recents. **Note**: `removeEntry()` bypasses the macOS Trash — there is no web API to move files to Trash. The confirm dialog is the only safety net.
- **Recent files**: last 5 opened notes shown at top of sidebar (pinned, not scrollable). Stored in `_notes.config.json`. Immediate-parent folder shown as a muted hint for disambiguation.
- **Folders collapsed by default**: avoids expanding 1000+ notes vault on load.
- **New note context**: "+ Note" creates the note in the same folder as the currently open note, or at root if nothing is open.
- **No file watcher**: external changes (Finder rename, another editor) are not detected. Manual ↻ refresh button re-scans the tree.
- **Separate IndexedDB**: DB name `notes-app` is distinct from kanban's `kanban-lite` — no cross-app handle pollution.

## Controls

- Sidebar: click file to open · click folder header to collapse/expand
- Recent section: top 5 last-opened notes, pinned above the tree
- `+ Note` button → prompts for name, creates `.md` in current folder (or root)
- `↻` button → re-scans the vault folder tree
- `✕` button (editor header) → delete current note (permanent, no Trash)
- Typing → auto-saves after 750 ms idle; preview updates live
- Drag the center divider → resize editor/preview split (20–80%)

## Open threads / next steps

### Near-term
- **Rename note**: double-click note title in editor header → inline rename. No native rename API; workaround is read → create new → write → delete old.
- **New folder**: "+ Folder" button alongside "+ Note" → `getDirectoryHandle(name, {create:true})`.
- **Search**: filter sidebar by filename as you type. Full-text search would require reading all files on each query — feasible but slow on large vaults; debounce + progressive scan.
- **Front matter display**: parse `title` / `creation_date` / `modification_date` from the `---` block and show them in the editor header.

### Medium-term
- **In-memory undo for delete**: keep deleted content in a variable for ~10 s with an "Undo" toast. The web has no API to move files to the OS Trash, so this is the only realistic safety net beyond the confirm dialog.
- **Word count / reading time** in the editor header.
- **Per-note last-modified** timestamp in sidebar (from `FileSystemFileHandle.getFile().lastModified`).
- **Drag-to-reorder or move** notes between folders — requires the read→create→delete workaround.
- **External change detection**: periodic re-scan (e.g., every 30 s) or a "Reload" banner when the file's `lastModified` differs from when it was last read.
