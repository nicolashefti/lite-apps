# Working notes — durable memory for dev & AI sessions

This folder is the project's working memory. It complements `CLAUDE.md` (stable instructions) with things that change as work progresses.

**Rules:**

- Read the latest note at the start of a session.
- One file per topic or session: `YYYY-MM-DD-topic.md`.
- Capture: gotchas discovered, decisions in flight (not yet ADR-worthy), open threads, known issues, ideas parked for later.
- Delete or fold notes that are no longer true — stale memory is worse than none.
- Anything that becomes a firm rule graduates to `CLAUDE.md`; anything that becomes a real decision graduates to `../decisions/`.
