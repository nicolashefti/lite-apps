## Full-text search detail

**Architecture**: in-memory index (`Map<path, content>`), built async after every `refreshTree`.

| Piece | Location | Role |
|---|---|---|
| `buildIndex(rootHandle, nodes)` | `fs.js` | Walks scanned tree nodes, reads each `.md` file, returns `Map`. Silently skips files removed between scan and read. |
| `renderSearchResults(results, query, container, onSelect)` | `tree.js` | Flat result list replacing the tree during search. Each card: note name (blue, truncated) + 140-char context snippet. Both highlight matched term with `<mark class="nt-highlight">`. |
| `runSearch(q)` / `exitSearch()` | `main.js` | `runSearch` hides recents, filters index, sorts (name matches first), calls `renderSearchResults`. `exitSearch` restores tree and recents. |
| `rebuildIndex()` | `main.js` | Token-guarded async rebuild; on completion re-runs active search if any. |
| `#nt-search` input | `notes.html` | 200ms debounce on `input`; Escape clears + exits search. |

**Index freshness**:
- Rebuilt on every `refreshTree` (new note, rename, delete, manual ↻, switch folder).
- `save()` updates `index.set(currentPath, content)` after each auto-save so the open note stays searchable while editing without a full rebuild.
- Token pattern (`indexToken`) discards stale builds if `refreshTree` fires again before a previous build completes.

**While indexing**: shows "Indexing…" placeholder; once built, automatically re-runs the current query.
